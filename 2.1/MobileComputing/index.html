Introduction to Mobile & Wireless Computing

Mobile Computing is a technology that allows transmission of data, voice, and video via a computer or any other wireless-enabled device without being connected to a fixed physical link.

Key Characteristics

  • Mobility — users can move and still communicate
  • Portability — devices are compact and easy to carry
  • Wireless Connectivity — no physical cabling required
  • Network Diversity — connects via cellular, Wi-Fi, Bluetooth, satellite, etc.
  • Context Awareness — devices can adapt based on location and environment
Limitations: Limited battery life, lower bandwidth compared to wired, security concerns, signal interference, and higher latency.

Mobile Computing vs Wireless Networking

Wireless networking refers to the medium (radio waves, microwaves) used to transmit data. Mobile computing is broader — it encompasses the devices, software, and infrastructure enabling untethered access to computing resources.

Evolution of Mobile Services

1G — First Generation (1980s)

  • Analog voice communication
  • Used FDMA (Frequency Division Multiple Access)
  • Poor battery life, large devices, no security

2G — Second Generation (1990s)

  • Digital voice (GSM — Global System for Mobile)
  • Used TDMA and CDMA
  • Introduced SMS text messaging
  • Better voice quality, capacity, and security

2.5G / GPRS

  • General Packet Radio Service
  • "Always on" data connectivity
  • Packet-switched data (not circuit-switched)

3G — Third Generation (2000s)

  • High-speed internet, video calls, mobile TV
  • Technologies: UMTS, WCDMA, HSDPA
  • Data rates: up to 2 Mbps stationary, 384 kbps moving

4G / LTE — Fourth Generation (2010s)

  • All-IP network (no circuit switching)
  • OFDMA and MIMO technologies
  • Peak data rates: 100 Mbps – 1 Gbps
  • Low latency (~50ms)

5G — Fifth Generation (2020s)

  • Extreme data rates (1–20 Gbps)
  • Ultra-low latency (<1ms)
  • Massive device connectivity (IoT)
  • Network slicing, beamforming, mmWave
Key Trend: Each generation brings higher data rates, lower latency, better spectral efficiency, and support for more devices.

Wireless Networks

Wireless Local Area Networks (WLAN)

Based on IEEE 802.11 standards (Wi-Fi). Operates in 2.4 GHz and 5 GHz bands. Range: ~100m indoors. Used for local connectivity in homes, offices, and public hotspots.

Wireless Personal Area Networks (WPAN)

Short-range networks (10–100m). Examples:

  • Bluetooth — v4.0+ (BLE low energy), v5.0 (2 Mbps, 240m range)
  • ZigBee — low data rate, low power, IoT applications
  • UWB — Ultra-Wideband, high precision positioning

Wireless Wide Area Networks (WWAN)

Cellular networks (2G–5G) covering vast geographic areas. Handled by mobile network operators with infrastructure of base stations and core networks.

Wireless Metropolitan Area Networks (WMAN)

Cover a city or campus. IEEE 802.16 (WiMAX) is a well-known standard. Bridges the gap between WLANs and WWANs.

Satellite Networks

Used for global coverage — GPS, satellite TV, broadband in remote areas. Categories: LEO, MEO, GEO (based on orbit altitude).

Comparison: WLAN = short range, high speed; WPAN = very short range, low power; WWAN = long range, carrier-managed; WMAN = city-wide, backhaul role.

Cellular Systems

A cellular network divides a region into cells, each served by a base station. Cells are grouped into clusters to reuse frequencies — the fundamental concept enabling massive capacity.

Key Concepts

  • Frequency Reuse — same frequencies reused in non-adjacent cells
  • Handoff / Handover — seamless transfer of a call from one cell to another
  • Cell Splitting — subdividing congested cells to increase capacity
  • Sectoring — using directional antennas to reduce interference

GSM Architecture

  • MS — Mobile Station (phone + SIM)
  • BSS — Base Station Subsystem (BTS + BSC)
  • NSS — Network & Switching Subsystem (MSC, HLR, VLR, AuC, EIR)
  • OSS — Operations Support Subsystem

Multiple Access Techniques

  • FDMA — Frequency Division Multiple Access
  • TDMA — Time Division Multiple Access
  • CDMA — Code Division Multiple Access
  • OFDMA — Orthogonal FDMA (used in 4G/5G)

Mobile IP & Mobility Management

Mobile IP allows devices to move between networks while maintaining a permanent IP address. Key entities:

  • Home Agent (HA) — located on the device's home network
  • Foreign Agent (FA) — located on the visited network
  • Care-of Address (CoA) — temporary address on the foreign network

How It Works

  1. Device moves to a foreign network and gets a CoA from the FA
  2. Device registers its CoA with the HA
  3. Packets sent to the home address are intercepted by the HA
  4. HA tunnels packets to the CoA (FA)
  5. FA delivers packets to the device (reverse path optional)
Note: Mobile IP is used for network-layer mobility. For application-layer mobility, protocols like SIP are used.

Mobility Management

  • Location Management — tracking device location (HLR/VLR in GSM)
  • Handoff Management — maintaining connectivity during movement
  • Types: Hard handoff (break-before-make) vs. Soft handoff (make-before-break)

Wireless Security

Wireless networks are inherently less secure than wired due to the broadcast nature of radio waves. Attacks can come from anywhere within range.

Common Threats

  • Eavesdropping — intercepting wireless signals
  • Man-in-the-Middle — attacker relays and alters communication
  • Replay Attack — capturing and retransmitting valid data
  • Denial of Service (DoS) — jamming or flooding
  • Rogue Access Points — unauthorized APs mimicking legitimate ones
  • War Driving — searching for open Wi-Fi networks

Security Mechanisms

  • WEP — Wired Equivalent Privacy (broken, deprecated)
  • WPA / WPA2 — Wi-Fi Protected Access (TKIP, AES-CCMP)
  • WPA3 — latest standard (SAE handshake, forward secrecy)
  • 802.1X / EAP — enterprise authentication
  • VPNs — secure tunnels over public networks
  • GSM Security — A3/A5/A8 algorithms (COMP128)
Best Practices: Use WPA3 or WPA2, disable WPS, change default credentials, use strong passphrases, enable network monitoring, and always use encryption (HTTPS, VPN).

Lesson 1: Introduction to Mobile Computing

Mobile computing enables users to access information and computing resources from anywhere using portable devices over wireless networks. It combines three key elements: mobile communication (the network), mobile hardware (the device), and mobile software (applications).

Key Concepts

  • Mobility — users move freely while staying connected
  • Portability — devices are lightweight and battery-powered
  • Wireless connectivity — communication via radio waves, infrared, or satellite
  • Context awareness — devices adapt to location, environment, and user state

Categories of Mobile Computing

  • Fixed and wired — traditional desktop computing
  • Mobile and wired — laptops connected via Ethernet
  • Fixed and wireless — desktop with wireless connection
  • Mobile and wireless — smartphone, tablet, laptop with cellular/Wi-Fi
Limitations: Limited battery life, lower bandwidth and higher latency than wired networks, security vulnerabilities, signal interference and fading, and higher power consumption for RF components.

Applications

Mobile computing is used in emergency services, healthcare (telemedicine), navigation (GPS), e-commerce (mobile banking), education (e-learning), transportation (fleet management), and entertainment (streaming, gaming).

Lesson 2: Mobile Computing Architecture

Mobile computing architecture defines the structure and interaction between mobile devices, networks, and backend services. It follows a layered approach similar to the OSI model but with adaptations for mobility.

Three-Tier Architecture

  • Presentation Tier (Client) — mobile device with user interface, handles display and input
  • Application Tier (Middleware) — application servers, logic processing, data transformation
  • Data Tier (Backend) — databases, file servers, enterprise systems

Mobile Client Types

  • Thin client — minimal processing; relies on server for computation (browser-based)
  • Thick client — significant local processing; app installed on device (native app)
  • Hybrid client — combines local and server processing (Progressive Web Apps)

Wireless Middleware

Wireless middleware bridges mobile clients and backend systems. It handles session management, data synchronization, compression, protocol translation, and offline support. Examples include WAP gateway and mobile application servers.

Key Challenges: Intermittent connectivity, varying bandwidth, device heterogeneity, and synchronization of data across multiple devices and the cloud.

Lesson 3: Wireless Communication Fundamentals

Wireless communication transfers information between devices without physical conductors. It uses electromagnetic waves — radio waves, microwaves, and infrared — to propagate signals through free space.

Electromagnetic Spectrum for Wireless

  • Radio waves (3 kHz – 300 MHz) — AM/FM radio, TV, cellular (penetrates buildings well)
  • Microwaves (300 MHz – 300 GHz) — Wi-Fi, Bluetooth, satellite, radar, 5G mmWave
  • Infrared (300 GHz – 400 THz) — remote controls, short-range line-of-sight

Signal Propagation Phenomena

  • Free-space path loss — signal attenuates with distance (proportional to d²)
  • Reflection — signal bounces off large surfaces (buildings, walls, ground)
  • Diffraction — signal bends around obstacles (edges of buildings, hills)
  • Scattering — signal disperses off rough surfaces or small objects
  • Multipath fading — multiple signal paths arrive at receiver with different phases, causing cancellation or reinforcement

Modulation Techniques

Modulation encodes digital data onto analog carrier waves. Common schemes used in mobile networks:

  • GMSK (Gaussian Minimum Shift Keying) — used in GSM
  • QPSK / 16-QAM / 64-QAM / 256-QAM — used in 3G/4G/5G, higher QAM = more bits/symbol
  • OFDM (Orthogonal Frequency Division Multiplexing) — divides spectrum into orthogonal subcarriers, resistant to multipath fading

Lesson 4: Cellular Networks — 1G, 2G, GSM

Cellular networks divide a geographic area into cells, each served by a Base Transceiver Station (BTS). The cellular concept, invented by Bell Labs in 1947, enables frequency reuse and massive capacity scaling.

1G — First Generation (1980s)

  • Analog voice using FDMA (AMPS, TACS, NMT standards)
  • No encryption — vulnerable to eavesdropping
  • Low capacity, large phones, poor battery life

2G — Second Generation (1990s)

  • Digital voice using TDMA and CDMA
  • GSM (Global System for Mobile Communications) became the global standard
  • Introduced SMS (Short Message Service), SIM cards, and digital encryption (A5 algorithm)
  • Data rates: 9.6 – 14.4 kbps (circuit-switched)

2.5G — GPRS & EDGE

  • GPRS (General Packet Radio Service) — packet-switched data, "always on", up to 114 kbps
  • EDGE (Enhanced Data rates for GSM Evolution) — up to 384 kbps using 8-PSK modulation

GSM Architecture

  • Mobile Station (MS) — Mobile Equipment (ME) + Subscriber Identity Module (SIM)
  • Base Station Subsystem (BSS) — BTS (radio transceivers) + BSC (Base Station Controller)
  • Network Switching Subsystem (NSS) — MSC (switching), HLR (home database), VLR (visitor database), AuC (authentication), EIR (equipment identity)
  • Operations Support Subsystem (OSS) — network management and maintenance

Lesson 5: 3G — UMTS Networks

3G networks, based on IMT-2000 standard, brought high-speed data and multimedia services to mobile devices. The dominant 3G technology is UMTS (Universal Mobile Telecommunications System).

UMTS Key Features

  • Data rates: up to 2 Mbps (stationary), 384 kbps (moving), 144 kbps (high-speed)
  • Uses WCDMA (Wideband Code Division Multiple Access) air interface
  • Circuit-switched for voice, packet-switched for data
  • Supports video calls, mobile TV, and high-speed internet

UMTS Architecture

  • User Equipment (UE) — mobile device
  • UTRAN (UMTS Terrestrial Radio Access Network) — Node B (base station) + RNC (Radio Network Controller)
  • Core Network — evolved from GSM NSS; includes SGSN (Serving GPRS Support Node) and GGSN (Gateway GPRS Support Node) for packet data

HSPA Evolution

  • HSDPA (High Speed Downlink Packet Access) — up to 14.4 Mbps downlink
  • HSUPA — up to 5.76 Mbps uplink
  • HSPA+ — up to 42 Mbps using MIMO and higher-order modulation
Key Standard: 3GPP Release 99 defined UMTS. Later releases (5, 6, 7) introduced HSDPA, HSUPA, and HSPA+.

Lesson 6: 4G — LTE and WiMAX

4G was defined by ITU's IMT-Advanced requirements: peak data rates of 100 Mbps (high mobility) to 1 Gbps (low mobility). Two technologies met these requirements: LTE-Advanced and WiMAX 802.16m.

LTE Key Features

  • All-IP network — no circuit-switched component; voice over LTE (VoLTE)
  • Uses OFDMA on downlink and SC-FDMA on uplink (better power efficiency)
  • MIMO (Multiple Input Multiple Output) — multiple antennas for spatial multiplexing
  • Peak rates: 300 Mbps downlink (LTE Cat 4), 1 Gbps (LTE-Advanced)
  • Low latency: ~50ms (LTE), ~10ms (LTE-Advanced)

LTE Architecture (EPS)

  • eNodeB — evolved Node B, combines BTS and BSC functions
  • MME (Mobility Management Entity) — signaling, authentication, mobility
  • S-GW (Serving Gateway) — user data routing and forwarding
  • P-GW (Packet Data Network Gateway) — connects to external IP networks
  • HSS (Home Subscriber Service) — evolved HLR

WiMAX (IEEE 802.16)

WiMAX (Worldwide Interoperability for Microwave Access) was developed as an alternative to LTE. It offered broadband wireless access over metropolitan areas (up to 50 km range). Mobile WiMAX (802.16e-2005) supported up to 100 Mbps. WiMAX lost the market to LTE and was largely phased out.

Lesson 7: 5G and Beyond

5G (Fifth Generation) is designed for extreme performance and massive connectivity. 3GPP Release 15 defined the initial 5G NR (New Radio) standard, with Release 16 and 17 adding enhancements.

5G Usage Scenarios

  • eMBB (Enhanced Mobile Broadband) — high-speed internet, 4K/8K streaming, VR/AR
  • URLLC (Ultra-Reliable Low-Latency Communications) — autonomous vehicles, remote surgery, industrial automation
  • mMTC (Massive Machine-Type Communications) — IoT, smart cities, sensor networks

Key 5G Technologies

  • mmWave (24–100 GHz) — large bandwidth but short range, needs beamforming
  • Beamforming — focuses signal directionally to a specific device
  • Network Slicing — multiple virtual networks on shared physical infrastructure
  • Massive MIMO — arrays of 64+, 128+, or 256 antenna elements
  • Small Cells — dense deployment of low-power base stations

5G Architecture

  • 5G NR gNB — next-generation base station
  • Service-Based Architecture (SBA) — cloud-native core network functions
  • AMF (Access and Mobility Management Function) — evolved MME
  • UPF (User Plane Function) — evolved S-GW/P-GW
  • Edge Computing — processing closer to the user for low latency
6G Vision: Research is underway for 6G (2030+), targeting terabit speeds, sub-ms latency, integrated sensing and communication, and AI-native network design.

Lesson 8: Wireless LANs and WPANs

Wireless LAN — IEEE 802.11 (Wi-Fi)

Wi-Fi is the dominant WLAN technology, operating in unlicensed ISM bands (2.4 GHz and 5 GHz). The 802.11 standard defines the MAC and PHY layers.

  • 802.11a — 5 GHz, OFDM, up to 54 Mbps
  • 802.11b — 2.4 GHz, DSSS, up to 11 Mbps
  • 802.11g — 2.4 GHz, OFDM, up to 54 Mbps (backward-compatible with b)
  • 802.11n (Wi-Fi 4) — 2.4/5 GHz, MIMO (up to 4×4), up to 600 Mbps
  • 802.11ac (Wi-Fi 5) — 5 GHz only, wider channels (80/160 MHz), MU-MIMO, up to 3.5 Gbps
  • 802.11ax (Wi-Fi 6) — 2.4/5 GHz, OFDMA, 1024-QAM, up to 9.6 Gbps

Wi-Fi MAC Protocol — CSMA/CA

Carrier Sense Multiple Access with Collision Avoidance: before transmitting, a station senses the channel. If busy, it waits a random backoff period. RTS/CTS (Request to Send / Clear to Send) mechanism reduces hidden node collisions.

Wireless PAN — Bluetooth (IEEE 802.15.1)

  • Operates in 2.4 GHz ISM band using frequency-hopping spread spectrum (FHSS)
  • Piconet: up to 8 devices (1 master, 7 slaves)
  • Bluetooth 4.0 — introduced BLE (Bluetooth Low Energy) for IoT
  • Bluetooth 5.0 — 2 Mbps data rate, 240m range, mesh networking

Other WPAN Technologies

  • NFC (Near Field Communication) — ~4 cm range for contactless payments
  • ZigBee (802.15.4) — low data rate, low power, mesh topology for home automation
  • UWB (Ultra-Wideband) — high precision positioning, short-range high-speed data

Lesson 9: Mobile IP and Mobility Management

Mobile IP (RFC 5944 / RFC 6275) allows devices to change network attachment points while maintaining a permanent IP address. It operates at the Network Layer (Layer 3) of the OSI model.

Mobile IP Entities

  • Home Agent (HA) — a router on the mobile node's home network that tunnels packets to the current location
  • Foreign Agent (FA) — a router on the visited network that provides routing services and a Care-of Address
  • Care-of Address (CoA) — temporary IP address assigned when the mobile node is on a foreign network
  • Correspondent Node (CN) — the peer device communicating with the mobile node

Mobile IP Operation

  1. Agent Discovery — mobile node detects whether it is on home or foreign network via ICMP Router Advertisement messages
  2. Registration — on a foreign network, the mobile node registers its CoA with its Home Agent using UDP
  3. Tunneling — HA intercepts packets destined for the mobile node's home address and encapsulates them (IP-in-IP, GRE) to the CoA
  4. Delivery — FA decapsulates and forwards to the mobile node, or the mobile node acts as its own FA (co-located CoA)

Mobility Management in Cellular Networks

  • Location Management — HLR/VLR databases track subscriber location; location update and paging procedures
  • Handoff Management — maintaining active connection during cell changes; hard handoff (break-before-make) vs. soft handoff (make-before-break)
  • Handoff Types: intra-BSC, inter-BSC, inter-MSC for GSM; X2 and S1 handovers for LTE
Note: Mobile IP handles macro-mobility (between networks). For micro-mobility within a network, protocols like Cellular IP, HAWAII, and Proxy Mobile IPv6 are more efficient.

Lesson 10: Mobile Security and Pervasive Computing

Wireless Security Threats

  • Eavesdropping — passive interception of radio signals using spectrum analyzers
  • Man-in-the-Middle (MitM) — attacker relays communication between two parties
  • Replay attack — capturing and retransmitting valid authentication packets
  • DoS/Jamming — overwhelming the channel with noise or traffic
  • Evil Twin / Rogue AP — fake access point mimicking a legitimate one

Wi-Fi Security Protocols

  • WEP — 64/128-bit RC4 encryption; broken in minutes (FMS attack)
  • WPA — TKIP with Michael integrity check; improved but still vulnerable
  • WPA2 (802.11i) — AES-CCMP encryption; current minimum standard
  • WPA3 (2018) — SAE handshake (replaces PSK), forward secrecy, 192-bit security mode, OWE (Opportunistic Wireless Encryption) for open networks

GSM Security

  • A3 algorithm — authentication (challenge-response using Ki on SIM)
  • A5/1, A5/2, A5/3 — stream ciphers for over-the-air encryption
  • A8 algorithm — generates session key (Kc) from Ki
  • Vulnerability: A5/1 can be cracked in minutes with rainbow tables

Pervasive (Ubiquitous) Computing

Pervasive computing, coined by Mark Weiser (1991), envisions computing embedded into everyday objects and environments, invisible to the user. Key characteristics:

  • Invisibility — computers disappear into the background
  • Context awareness — systems sense and respond to environment
  • Ad-hoc networking — devices discover and communicate spontaneously
  • Smart environments — smart homes, smart cities, smart healthcare
Example: A smart refrigerator that detects grocery depletion, checks inventory via RFID, and automatically places an online order — all without user intervention.
