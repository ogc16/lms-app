================================================================================
                MAUA ART GALLERY MANAGEMENT SYSTEM
                         WRITTEN USE CASES
================================================================================

USE CASE 1: Browse Artworks
------------------------------------------------
Use Case ID:      UC01
Actor:            Ordinary Client, Artist, Collector
Description:      User views the gallery's collection of artworks with filtering
                  and search capabilities.
Precondition:     User is on the gallery homepage.
Main Flow:
  1. User navigates to the Browse Artworks page.
  2. System displays a list of available artworks with thumbnails.
  3. User can filter by medium, artist, price range, or year.
  4. User selects an artwork to view its full details.
  5. System displays artwork details (title, artist, dimensions, medium,
     price, provenance, description).
  6. User may click on "View Provenance" to see ownership history.
Postcondition:    User has browsed the gallery collection.
Alternative Flow:
  3a. User searches by keyword; system returns matching results.


USE CASE 2: Register
------------------------------------------------
Use Case ID:      UC02
Actor:            Ordinary Client
Description:      A new user creates an account on the system.
Precondition:     User is not logged in.
Main Flow:
  1. User clicks "Register" on the login page.
  2. System displays registration form (name, email, password, phone).
  3. User fills in details and submits.
  4. System validates input and checks for duplicate email.
  5. System creates user account as Ordinary Client.
  6. System sends confirmation email.
  7. System displays success message and redirects to login.
Postcondition:    New Ordinary Client account is created.
Alternative Flow:
  4a. Email already exists: system displays error message.
  4b. Required fields missing: system highlights missing fields.


USE CASE 3: Login
------------------------------------------------
Use Case ID:      UC03
Actor:            Ordinary Client, Artist, Collector, Admin
Description:      Registered user authenticates into the system.
Precondition:     User has a registered account.
Main Flow:
  1. User navigates to login page.
  2. System displays login form (email, password).
  3. User enters credentials and submits.
  4. System validates credentials.
  5. System grants access and redirects to appropriate dashboard.
Postcondition:    User is logged into the system.
Alternative Flow:
  4a. Invalid credentials: system displays error and allows retry.
  4b. Unverified account: system prompts user to verify email first.
Extends:          UC02 (Register) – registration may be followed by login.


USE CASE 4: Manage Profile
------------------------------------------------
Use Case ID:      UC04
Actor:            Artist
Description:      Artist updates their personal profile information.
Precondition:     Artist is logged in.
Main Flow:
  1. Artist navigates to Profile settings.
  2. System displays current profile details.
  3. Artist updates fields (bio, CV, website, social media links).
  4. System validates and saves changes.
  5. System confirms successful update.
Postcondition:    Artist profile is updated.
Alternative Flow:
  3a. Artist uploads new CV document; system validates file type/size.


USE CASE 5: Add Artwork
------------------------------------------------
Use Case ID:      UC05
Actor:            Artist
Description:      Artist adds a new artwork to their portfolio.
Precondition:     Artist is logged in.
Main Flow:
  1. Artist navigates to "My Portfolio" and clicks "Add Artwork".
  2. System displays artwork submission form.
  3. Artist enters details: title, dimensions, medium, price, year,
     description, and uploads image.
  4. System validates the data.
  5. System saves artwork with status "AVAILABLE".
  6. System confirms success and displays new artwork.
Postcondition:    Artwork is added to the gallery and artist's portfolio.
Included In:      UC08 (Manage Artist Portfolio)


USE CASE 6: Edit Artwork
------------------------------------------------
Use Case ID:      UC06
Actor:            Artist
Description:      Artist modifies an existing artwork's details.
Precondition:     Artist is logged in and artwork exists.
Main Flow:
  1. Artist navigates to artwork in portfolio and clicks "Edit".
  2. System displays current artwork details.
  3. Artist modifies fields.
  4. System validates and saves changes.
  5. System confirms update.
Postcondition:    Artwork details are updated.
Alternative Flow:
  3a. Artist changes status to "SOLD" and enters sale details.
Included In:      UC08 (Manage Artist Portfolio)


USE CASE 7: Remove Artwork
------------------------------------------------
Use Case ID:      UC07
Actor:            Artist
Description:      Artist removes an artwork from their portfolio.
Precondition:     Artist is logged in and artwork exists.
Main Flow:
  1. Artist navigates to artwork in portfolio and clicks "Remove".
  2. System prompts for confirmation.
  3. Artist confirms deletion.
  4. System removes artwork and updates portfolio.
  5. System confirms removal.
Postcondition:    Artwork is removed from the system.
Alternative Flow:
  3a. Artist cancels; no changes made.
Included In:      UC08 (Manage Artist Portfolio)


USE CASE 8: Manage Artist Portfolio
------------------------------------------------
Use Case ID:      UC08
Actor:            Artist
Description:      Artist manages their complete portfolio including artworks,
                  exhibitions, and sales history.
Precondition:     Artist is logged in.
Main Flow:
  1. Artist navigates to Portfolio Dashboard.
  2. System displays summary of artworks, exhibitions, and sales.
  3. Artist may add, edit, or remove artworks (UC05-UC07).
  4. Artist may manage exhibition participation.
Postcondition:    Portfolio is updated.
Includes:         UC05 (Add Artwork), UC06 (Edit Artwork), UC07 (Remove Artwork)


USE CASE 9: Purchase Artwork
------------------------------------------------
Use Case ID:      UC09
Actor:            Collector
Description:      Collector purchases an available artwork.
Precondition:     Collector is logged in and artwork is available.
Main Flow:
  1. Collector browses to an artwork and clicks "Purchase".
  2. System displays purchase confirmation with price and terms.
  3. Collector confirms purchase.
  4. System processes payment.
  5. System records transaction and updates artwork status to "SOLD".
  6. System adds provenance record with new owner.
  7. System confirms purchase and sends receipt.
Postcondition:    Transaction is completed; artwork ownership transferred.
Alternative Flow:
  4a. Payment fails: system displays error and allows retry.


USE CASE 10: View Purchase History
------------------------------------------------
Use Case ID:      UC10
Actor:            Collector
Description:      Collector views their past artwork purchases.
Precondition:     Collector is logged in.
Main Flow:
  1. Collector navigates to "My Purchases".
  2. System displays list of past transactions with artwork details,
     date, and amount.
  3. Collector may click on a transaction for full details.
Postcondition:    Purchase history is displayed.


USE CASE 11: View Provenance
------------------------------------------------
Use Case ID:      UC11
Actor:            Artist, Collector
Description:      User views the full provenance history of an artwork.
Precondition:     User is logged in.
Main Flow:
  1. User navigates to an artwork's details page.
  2. User clicks "View Provenance".
  3. System displays chronological list of ownership transfers,
     sale prices, and dates.
Postcondition:    Provenance history is displayed.
Included In:      Browse Artworks (when viewing artwork details)


USE CASE 12: Manage Exhibitions
------------------------------------------------
Use Case ID:      UC12
Actor:            Admin, Artist
Description:      Admin schedules gallery exhibitions; Artist participates.
Precondition:     User is logged in with appropriate role.
Main Flow:
  1. Admin creates new exhibition with date, location, theme.
  2. System saves exhibition and makes it visible to artists.
  3. Artist submits artworks for consideration.
  4. Admin approves/rejects submissions.
  5. System updates exhibition roster.
Postcondition:    Exhibition is scheduled and populated.
Alternative Flow:
  1a. Admin edits or cancels existing exhibition.


USE CASE 13: Manage Users
------------------------------------------------
Use Case ID:      UC13
Actor:            Admin
Description:      Admin manages all user accounts.
Precondition:     Admin is logged in.
Main Flow:
  1. Admin navigates to User Management.
  2. System displays list of all users with roles.
  3. Admin can activate, deactivate, or change user roles.
  4. System updates user records.
Postcondition:    User accounts are updated.


USE CASE 14: View Sales Dashboard
------------------------------------------------
Use Case ID:      UC14
Actor:            Admin
Description:      Admin views sales analytics and reports.
Precondition:     Admin is logged in.
Main Flow:
  1. Admin navigates to Sales Dashboard.
  2. System displays revenue charts, top-selling artists,
     popular mediums, and transaction summaries.
  3. Admin may filter by date range.
Postcondition:    Sales analytics displayed.


USE CASE 15: Generate Reports
------------------------------------------------
Use Case ID:      UC15
Actor:            Admin
Description:      Admin generates system reports for analysis.
Precondition:     Admin is logged in.
Main Flow:
  1. Admin navigates to Reports.
  2. System displays report type options (sales, users, artworks).
  3. Admin selects report type and parameters.
  4. System generates report in PDF/Excel format.
  5. System offers download or email options.
Postcondition:    Report is generated and delivered.
